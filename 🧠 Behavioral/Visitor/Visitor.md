# Паттерн проектирования: Посетитель (Visitor)

**Посетитель** — это поведенческий паттерн, который позволяет добавить новую операцию для целой иерархии классов, не изменяя код этих классов.



---

## В чем проблема?
Представь, что у нас есть классы `Shawarma` и `Drink`.
1. Сначала нам нужно посчитать калории. Мы могли бы добавить метод `getCalories()` прямо в эти классы.
2. Потом нам нужно выгрузить данные в XML. Добавляем метод `toXml()`.
3. Потом нужно сделать JSON, потом CSV...

В итоге классы еды превращаются в мусорку с методами, которые не имеют отношения к самой еде. **Visitor** решает эту проблему, вынося "посторонние" операции в отдельные классы.

---

## Разбор реализации

### 1. Интерфейс Посетителя (Visitor)
Определяет контракт: "Я умею работать с Шаурмой и с Напитком".
```java
interface Visitor {
    void visit(Shawarma shawarma);
    void visit(Drink drink);
}
